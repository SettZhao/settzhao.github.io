# 菜谱系统使用说明 📖

## ⚡ 快速开始（推荐）

现在添加菜谱超级简单！不需要创建HTML文件，只需要：

### 1️⃣ 编辑数据文件
打开 `recipes-data.js`，在数组中添加你的菜谱数据：

```javascript
{
    id: "caipin-pinyin",              // 菜品拼音ID
    name: "你的菜名",
    emoji: "🍖",
    subtitle: "一句话介绍",
    time: "30分钟",
    servings: "2-3人份",
    difficulty: "★★☆",
    description: "详细描述...",
    tags: ["标签1", "标签2"],
    image: "images/你的菜名.jpg",
    
    ingredients: [
        { name: "食材", amount: "用量" }
    ],
    steps: [
        "步骤描述..."
    ],
    tips: [
        "小贴士..."
    ]
}
```

### 2️⃣ 添加图片（可选）
把菜品照片放到 `images/` 文件夹，系统会自动加载。

### 3️⃣ 刷新页面
打开 `index.html`，新菜谱自动显示！

👉 **详细教程请看：[使用指南.md](使用指南.md)**

## 🎯 两种使用方式对比

### 方式一：数据驱动（推荐⭐）
- 编辑 `recipes-data.js` 添加数据
- 页面自动生成
- 适合：批量管理、快速添加

### 方式二：手动创建（旧版）
- 复制 `_模板.html` 创建新页面
- 手动填写所有内容
- 手动更新列表页链接
- 适合：需要高度自定义的特殊页面

**建议：**优先使用方式一，除非有特殊定制需求。

## ❓ 常见问题

**Q: 旧的HTML文件还需要吗？**
A: 不需要了！可以删除 `红烧肉.html`、`番茄炒蛋.html` 等旧文件，以及 `_模板.html`。

**Q: 如何删除菜谱？**
A: 在 `recipes-data.js` 中删除对应的数据对象即可。

**Q: 可以修改菜谱信息吗？**
A: 直接在 `recipes-data.js` 中修改对应数据，刷新页面即可看到更新。

**Q: 图片必须要吗？**
A: 不是！没有图片会显示emoji，效果也很好。

**Q: 能同时显示图片和emoji吗？**
A: 目前只能二选一。有图片优先显示图片，图片加载失败才显示emoji。
